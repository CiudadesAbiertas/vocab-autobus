El vocabulario del servicio público de transporte de autobuses urbano utiliza fundamentalmente los módulos del vocabulario Transmodel, se apoya en otros vocabularios (sosa, geo, kos y organization) además de crear algunas clases y propiedades ad hoc.

En concreto, de Transmodel se han reutilizado los siguientes vocabularios:

  • Commons, tmcommons, para representar algunos conceptos y propiedades comunes de los módulos de Transmodel.
  • Journeys, tmjourneys, para la información relativa a viajes.
  • Organisations, tmorg, para representar la información relativa a las organizaciones que participan en el sistema de transporte público de autobuses urbanos.

Los otros vocabularios utilizados han sido:

  • Para las taxonomías se ha empleado SKOS (Simple Knowledge Organization System), en concreto:
    • tmkoscommons para taxonomías comunes.
    • tmkosjourney para taxonomías de viajes.
  • Para la representación geográfica de los datos se han reutilizado los siguientes vocabularios:
    • geo, para las propiedades de latitud y longitud en WGS84.
    • geo_core, para las propiedades de latitud y longitud en ETRS89.
    • geosparql para la representación geográfica de las tres entidades.
    • sf (Simple Features Geometries), para la representación del punto sf:Point.
  • Para algunos valores de los autobuses o de las rutas se ha reutilizado el vocabulario de sensores SOSA (Semantic Sensor Network Ontology).
  • Se ha reutilizado el vocabulario de Tráfico estraf para incidencias.
  • Para la dirección de la parada se ha reutilizado el vocabulario de callejero escjr.
  • Se ha empleado el vocabulario org para las organizaciones de transporte.
